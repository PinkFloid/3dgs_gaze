# 实验方案 v1 —— 注视指代跨房间取物(目标 ICRA 2027,截稿 2026-09-15)

> 声明(三支柱):可穿戴注视消解口语指代 / 持久全局地图消解且**目标在机器人视野外** / 跨房间真实移动取物。
> 实验的任务:把每根支柱变成一个数,并证明去掉任何一根都不行。
> 纪律:**指标与失败分类在采集前定死(本文档即预注册);所有 trial 全记录、全报告,不许挑。**

---

## 0. 场地与真值协议(所有实验共用)

- **布局**:房间1 = 用户+目标物体(含同类干扰物);房间2 = 狗的停放点。狗执行时穿过门/走廊进入房间1,取物后送回用户处。画平面图(论文 Fig.3):两房尺寸、门宽、tag 分布、物体桌位置、停放点。
- **真值**:每个 trial 由指令卡指定目标(如"左起第2只杯子"),物体位置编号与地图实例 id 的对应表提前拍照存档。系统选择 = `attention.binding` 日志里的实例 id;判对 = 与卡一致。
- **记录**:每 trial 落盘 raw gaze 流(--raw-log)、events.jsonl、skill.status、外部机位视频。全部可回放 → E4 免费。
- **同类物体**:同款杯子 ≥5 只(同色同形,无任何标记——与 iBotAssistant 的 AprilTag 方案划清界限的物证)。

## 1. E1|指代消解准确率(论文核心,不需要真狗)

**问题**:语言无法区分同类多实例时,注视能否选对?

- **自变量**:
  - 同类实例数 N ∈ {2, 3, 5}(间距固定 0.5m,用户距物 ~2m);
  - 相邻实例**角间隔**(N=3 固定,间距 {0.25, 0.5, 1.0}m × 距离 {1.5, 3}m 组合出角间隔梯度)。
- **关键图(Fig.4)**:准确率 vs **角间隔**——把距离和间距折叠成一根曲线,并把标定的锥 σ(≈1°)画成竖线。这张图把"为什么能分对"从工程叙事变成几何解释,是全文最硬的一张。
- **流程**:指令卡指定目标 → 用户注视并说"把这个杯子拿来" → 系统绑定 → 记录。每格 15-20 trial,总计 ~90-100 trial(约 2 个半天)。
- **基线**(同一批 trial 上评):
  1. 纯语言(类别→类内均匀随机):解析上就是 1/N,同时用 LLM 按指令原文选一次做实测版;
  2. "离用户最近的同类"启发式(有时会赢,诚实报告)。
- **指标**:top-1 准确率(Wilson 95% CI)、绑定分数余量(第一名减第二名)、与基线的 McNemar 检验。

## 2. E2|端到端跨房取物成功率(需要真狗)

- **任务**:狗在房间2,用户在房间1 发指令,狗进房→取→送回用户处。
- **规模**:K ≥ 25 trial,≥2 类目标(杯 + 另一类),2-3 种物体摆位;每 trial 硬性重置(狗回停放点、物体归位)。
- **指标**:
  - 端到端成功率 + 分阶段成功率:指代消解 / 导航到接近位姿(板系残差 <0.3m)/ 抓取 / 送达;
  - 时延分解表:说完→dispatch、dispatch→到位、到位→抓稳、总时长;
  - 人工干预计数(0 干预才算 success,干预的 trial 单列)。
- **失败分类(预注册,失败必归一类)**:定位空窗 / 选错相邻实例 / 绑定窗错过(说话时没看)/ ASR 错 / 导航失败 / 抓取失败 / 通信超时。

## 3. E3|视野外不变性(差异化支柱的直接证据)

- **设计**:同一取物任务,两个条件各 10-12 trial:
  - (a) 命令时目标在狗的相机视野内(狗与用户同房间、面向物体);
  - (b) 命令时狗在房间2(标准跨房条件)。
- **预期**:我们的系统两条件成功率**无差**——地图中介让机器人视野无关紧要;这个"不变性"就是支柱二的量化形态。对比叙述:视野依赖方法(FAM-HRI 类,需目标在机器人当前图像中)在 (b) 结构性不可用,引文说明即可,不必复现。
- 顺带用 (a)/(b) 差分把"导航难度"从"消解能力"里剥离出来。

## 4. E4|离线消融(全部回放,零真机成本)

在 E1/E2 的录像上重放扫参:

| 消融 | 扫什么 | 回答什么 |
|---|---|---|
| 眼-声绑定窗 | 窗口起点/宽度(0~3s,说指代词前) | 选定窗口有据可依;联上 eye-voice span 文献 |
| 类别过滤 | 开/关 | 名词类别贡献多少准确率 |
| 在线 bias 戳 | 开/关(回放时禁用戳) | 你的标定机制值多少个百分点——把管线设计正当化 |
| 票面门槛 | vote_share 0.3-0.7 | 稳健性 |

## 5. E5|小规模跨用户(可选加分项,时间允许才做)

4-6 名未接触过系统的被试,各自 gaze 标定后做 N=3 的 E1(每人 10 trial)。只报指代准确率,不做主观量表。价值:把"单作者可用"升级成"人皆可用";成本:一天。截稿前两周若进度紧,**砍它,不砍 E1-E4**。
