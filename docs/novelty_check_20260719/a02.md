### Closest 3

1. **[EgoSpot](https://arxiv.org/abs/2306.02393).** For pillar (a), it has genuine HoloLens eye gaze and voice, but voice controls modes/actions while gaze selects a position—it does not resolve the “this” in a spoken command to an object instance. For (b), Azure Spatial Anchors provide a shared frame and the gaze ray intersects the HoloLens mesh, but that mesh is not a persistent global instance map and the referent-outside-robot-view condition is not demonstrated. For (c), it uses the claimed Spot-style quadruped-plus-arm and performs navigation and arm control, but only as separate local tasks, not autonomous cross-room fetch. It is therefore the closest headset-and-hardware architecture, while missing the map-mediated deictic-fetch chain.

2. **[HoloSpot](https://holospot.github.io/).** For (a), it uses a HoloLens but object reference is explicit hand drag-and-drop, not wearable eye-gaze deixis; speech is limited to interface utilities. For (b), it comes closest: a pre-scanned metric room is persistently represented, segmented into instances, and shared with the robot by instance index and coordinates, so selection does not depend on the robot’s live camera. For (c), Spot navigates, grasps, and places the selected object, but demonstrations remain within one scanned room and do not establish that the robot began in another room or had never observed the referent during interaction. Replacing its hand interaction with gaze-plus-speech and extending the demonstrated route across rooms would approach the claim directly.

3. **[FAM-HRI](https://arxiv.org/abs/2503.16492).** For (a), this is essentially an exact match: Meta Aria is a wearable eye tracker and microphone, speech carries the task, and gaze resolves the referenced object. For (b), however, ORB-SLAM is used to compensate wearer motion and aggregate gaze in the egocentric stream; the selected object is then feature-matched into the robot’s current RGB-D image rather than looked up in a persistent global instance map. The experimental setup explicitly gives the robot camera clear visibility of the tabletop workspace. For (c), execution is by a fixed Franka arm, with no mobile navigation or cross-room retrieval.

### Verdict

CLAIM SURVIVES — In the live, verifiable 2020–2026 public record searched, no single system demonstrates all three pillars in one end-to-end trial. FAM-HRI/Glance-Say satisfy wearable-gaze-plus-speech deixis, while HoloSpot, EgoSpot, Reality Promises, and MINA cover different subsets of shared mapping and mobile manipulation; none gaze-resolves a mapped instance while the robot is in another room and then performs cross-room fetch. This is an evidence-bounded literature verdict, not proof against unpublished, proprietary, or patent-only systems.
