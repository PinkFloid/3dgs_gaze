### Closest 3

1. **[iBotAssistant](https://doi.org/10.3390/s26061958).** Pillar (a) is substantially met: a Pupil Core wearer speaks a request such as “Get me a lumber,” while gaze selects the tagged instance. For pillar (b), however, the gaze-to-object mapper outputs an AprilTag ID; the robot’s pre-created 2D map localizes the robot but is not the medium that grounds gaze to an object world coordinate. The paper explicitly places the worker and robot in a shared workspace with one storage area and does not demonstrate a different-room referent. Pillar (c) is partially met through autonomous mobile retrieval and delivery, but not cross-room fetch mediated by a persistent global 3D object map.

2. **[FAM-HRI](https://arxiv.org/abs/2503.16492).** Pillar (a) is met almost verbatim: Meta Aria supplies gaze and speech, speech describes the action, and gaze disambiguates the referenced object. Pillar (b) fails because the chosen object is transferred by matching features into the robot’s live camera image; ORB-SLAM3 supports temporal gaze reprojection but is not a persistent shared object map, and the referent must be visible from the robot workspace. Pillar (c) is limited to a stationary Franka performing tabletop manipulation, with no room-to-room navigation or fetch.

3. **[Glance-Say](https://arxiv.org/abs/2603.06121).** Pillar (a) is met through Meta Aria gaze selection and spoken action commands or confirmations. Pillar (b) is the closest partial match: the robot explores its local workspace to form object point clouds and a topology graph with 3D positions, then localizes the eyewear view against that representation; nevertheless, no persistent building-scale map or target outside the robot workspace is demonstrated. Pillar (c) is absent because execution uses a fixed Franka over a tabletop rather than a mobile manipulator performing cross-room fetch.

### Verdict

**CLAIM SURVIVES.** Through 19 July 2026, no verified work found combines wearable-gaze deixis, grounding through a persistent metric global object-instance map while the referent is outside the robot’s view, and cross-room mobile fetch. The closest mobile precedent, iBotAssistant, already makes “wearable gaze + speech + mobile retrieval” too broad to claim alone; the defensible novelty is the full global-map-mediated, different-room conjunction.
