### Closest 3

1. **[Gaze2Act](https://arxiv.org/abs/2605.30282).** Pillar (a) is nearly met: Meta Aria gaze supplies the target while language supplies the action, and the evaluation explicitly contains near-identical instances. It nevertheless uses a voice keyword to sample gaze rather than demonstrating a spoken deictic such as “this cup.” Pillar (b) fails because gaze is semantically matched against masks in the robot’s live camera, not raycast through a persistent global map; an out-of-view referent cannot be selected. Pillar (c) also fails because the Unitree G1 performs local tabletop manipulation without mobile navigation or fetch.

2. **[OK-Robot](https://www.roboticsproceedings.org/rss20/p091.html).** Pillar (a) fails completely: target grounding is language-only. Pillar (b) is unusually close because an external iPhone scan creates a global metric semantic memory, allowing the robot to receive a world location for something it did not itself map. Multiple similar instances are not robustly identified: the system uses the best CLIP voxel or a crude relational proximity rule. Pillar (c) includes real navigation, grasping, and delivery, but the published trials do not establish the claimed different-room start/referent configuration.

3. **[BUMBLE](https://arxiv.org/abs/2410.06237).** Pillar (a) fails because commands are free-form language without wearable gaze or deictic binding. Pillar (b) also fails: its topological map stores landmarks, while its two memories store execution history and previous errors—not persistent object instances and coordinates; the target is selected from live robot vision after reaching a likely room. Its current-view Set-of-Mark/VLM mechanism can use visual attributes, but it does not identify which otherwise identical instance a person meant. Pillar (c), however, is fully threatening: it demonstrates physical retrieval across rooms and floors, including doors and elevators.

### Verdict

CLAIM SURVIVES — As of 19 July 2026, no publicly documented system found combines wearer-eye-gaze resolution of a spoken deictic, a persistent shared global metric instance map resolving the referent while it is outside robot view, and physical different-room mobile fetch. The decisive split is that gaze systems require live robot-view correspondence, whereas memory-fetch systems select by text similarity, visual attributes, or relational descriptions—and one prominent system accepts any same-category instance. I interpret “never in the robot’s field of view” as “not before or during reference resolution”; taken literally through the eventual vision-guided grasp, that phrase would be impossible.
